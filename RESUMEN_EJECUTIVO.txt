1. CÓMO FUNCIONA LA BÚSQUEDA DE "PIANO"
═══════════════════════════════════════════════════════════════════════════════

ENTRADA:
  Usuario: "Quiero un piano"
  ↓

PROCESO (6 pasos):
  
  Paso 1: classifyIntent()
    ├─ Pregunta a OpenAI: "¿Cuál es la intención?"
    └─ Retorna: {intent: "consulta_producto", entities: {product: "piano"}}
  
  Paso 2: Get Active Products
    ├─ Query: SELECT name FROM products WHERE is_active=true
    └─ Retorna: ["Piano Profesional", "Piano Digital", "Guitarra", ...]
  
  Paso 3: searchRelevantProducts()
    ├─ Pregunta a OpenAI: "¿Qué productos de la lista busca el cliente?"
    ├─ Normaliza respuesta (elimina caracteres especiales)
    ├─ Valida coincidencias
    └─ Retorna: ["Piano Profesional", "Piano Digital"]
  
  Paso 4: Query BD
    ├─ SELECT * FROM products WHERE name IN (...) AND is_active=true
    └─ Retorna: Objetos de producto con precio, descripción, etc.
  
  Paso 5: orchestrate()
    ├─ Pregunta a OpenAI: "¿Qué agente debe responder?"
    └─ Retorna: "interes_producto"
  
  Paso 6: generateResponse()
    ├─ Pregunta a OpenAI: "Responde sobre estos pianos: [contexto]"
    └─ Retorna: "¡Excelente! Tenemos dos opciones: Piano Profesional $5999..."

SALIDA:
  Respuesta con productos y precios al usuario
  ✓ Éxito

2. ¿POR QUÉ A VECES FALLA?
═══════════════════════════════════════════════════════════════════════════════

5 PROBLEMAS PRINCIPALES:

PROBLEMA #1: Producto NO existe en BD
  ├─ Síntoma: Query retorna 0 filas
  ├─ Ubicación: Tabla products
  ├─ Verificar: SELECT COUNT(*) FROM products WHERE name ILIKE '%piano%'
  └─ Solución: Importar productos (JSON, CSV, o WooCommerce)

PROBLEMA #2: Producto existe pero isActive = FALSE
  ├─ Síntoma: Existe pero no aparece en búsquedas
  ├─ Ubicación: Column products.is_active
  ├─ Verificar: SELECT is_active FROM products WHERE name ILIKE '%piano%'
  └─ Solución: UPDATE products SET is_active=true WHERE name ILIKE '%piano%'

PROBLEMA #3: LLM NO reconoce el nombre del producto
  ├─ Síntoma: LLM retorna "none" aunque exista
  ├─ Ubicación: searchRelevantProducts() en aiService.ts línea 101-108
  ├─ Causa: Nombre muy diferente (ej: "Teclado de 88 teclas" vs "piano")
  ├─ Verificar: Ver si producto es intuitivo para cliente
  └─ Solución: Renombrar producto o agregar sinónimos

PROBLEMA #4: Normalización de string falla
  ├─ Síntoma: Caracteres especiales en nombre del producto
  ├─ Ubicación: searchRelevantProducts() línea 118-125
  ├─ Ejemplo: "Piano—Digital" (guión largo) → "pianodigital" (sin espacio)
  ├─ Verificar: Nombres con caracteres: — / – \ etc.
  └─ Solución: Limpiar nombres en BD

PROBLEMA #5: Base de datos desconectada
  ├─ Síntoma: "Error fetching product context"
  ├─ Ubicación: DATABASE_URL en .env
  ├─ Verificar: psql $DATABASE_URL -c "SELECT 1"
  └─ Solución: Verificar conexión PostgreSQL
